📋 项目概述

什么是 Kealin AI Novels？

Kealin AI Novels 是一个基于大语言模型的小说创作辅助工具。它不是要替代作者，而是帮助作者更高效地完成创作。核心理念是：

  • 反AI味：生成的内容要像人类写的，而不是AI模板
  • 作者主导：AI是工具，作者是创作者，作者拥有完全的控制权
  • 结构化创作：从大纲到章节到正文，层层递进，确保逻辑连贯
  • 记忆连贯：解决AI写作中常见的"失忆"问题，保持长篇小说的连贯性
核心特性：永久记忆系统 + 滑动窗口上下文 + 反AI味提示词 + 段落级微观控制

🎨 前端功能详解

📝 创作工作台

核心创作区域，包含：

  • 故事大纲：AI生成或手动编写
  • 章节管理：从大纲拆分章节，支持拖拽排序
  • 正文生成：单章或批量生成正文
  • 实时字数统计：显示大纲字数/正文字数

⚙️ 基础设定

定义小说的基本要素：

  • 小说背景：世界观、时代背景
  • 人物设定：主角、配角的性格能力
  • 角色关系：人物之间的关系
  • 核心剧情：故事主线、核心冲突
  • 写作风格：文风、节奏、语言特点

👤 角色管理

结构化角色档案：

  • 名称、外貌、性格
  • 口头禅、说话风格
  • 背景故事
  • 角色关系网络

🧠 永久记忆

作者主动添加的长期记忆：

  • 重要设定和规则
  • 需要保持一致的细节
  • 每次生成时自动注入AI上下文
  • 支持增删改查

🌍 世界观笔记

记录世界观相关设定：

  • 地理、历史、势力
  • 魔法/科技体系
  • 社会制度、规则

💡 灵感笔记

随时记录创作灵感：

  • 快速记录想法
  • 支持标签分类
  • 可插入到章节正文

🎯 AI助手

右侧AI对话助手：

  • 创作建议和灵感
  • 问题解答
  • 对话历史搜索

📊 写作统计

追踪写作进度：

⚙️ 后端服务

后端架构

后端采用极简设计，仅负责API代理，不存储任何业务数据。所有数据都存储在用户浏览器的localStorage中。

核心API端点

端点 方法 说明
/gen POST 生成文本（流式响应），接收prompt参数
/api/health GET 健康检查，返回服务状态和版本号
/ GET 主页面（渲染index.html模板）

配置管理

通过 .env 文件配置：

# 主模型配置 PRIMARY_API_KEY=your_api_key PRIMARY_API_ENDPOINT=https://api.openai.com/v1 PRIMARY_MODEL=gpt-4 # 辅助模型配置（用于AI助手等功能） SECONDARY_API_KEY=your_api_key SECONDARY_API_ENDPOINT=https://api.openai.com/v1 SECONDARY_MODEL=gpt-3.5-turbo # 服务配置 PORT=20000 HOST=0.0.0.0 DEBUG=true

🧠 记忆系统详解

记忆系统架构

本项目实现了分层记忆系统，解决AI写作中的"失忆"问题：

短期记忆（滑动窗口）

生成正文时自动注入前面章节的内容：

  • 最近10章的内容作为上下文
  • 较长章节只保留开头+结尾
  • 确保AI了解前文剧情发展
实现方式：getPreviousChaptersContext(idx, 10)

永久记忆（用户主动）

作者手动添加的重要设定：

  • 角色核心性格设定
  • 世界观不可违反的规则
  • 写作风格特殊要求
  • 需要长期保持的伏笔
实现方式：getPermanentMemoryContext()

🔌 API文档

POST /gen

生成文本的主要API端点，支持流式响应。

请求体

{ "prompt": "你是一个网文写手..." }

响应

返回SSE流式响应，每个事件格式：

data: {"text": "生成的文本片段"}

错误处理

最多重试3次，指数退避。失败时返回：

data: {"error": "错误信息"}

GET /api/health

健康检查端点，返回服务状态。

响应

{ "status": "ok", "models": { "primary": { "configured": true, "model": "gpt-4", "endpoint": "https://api.openai.com/v1" }, "secondary": { "configured": true, "model": "gpt-3.5-turbo", "endpoint": "https://api.openai.com/v1" } }, "version": "3.0.0" }
