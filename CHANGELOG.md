# Changelog — Kealin AI Novels 超级进化记录

## v4.0.0 - 无限画布 · 专业提示词 (2026-07-10)

> 三大核心升级：无限画布引擎、专业编剧提示词、架构优化

### 🎨 无限画布引擎

- **全新 `canvas-engine.js` 模块**（663行）：零依赖的无限画布引擎
- 鼠标滚轮缩放（以鼠标位置为中心），支持 20%-300% 缩放范围
- 拖拽空白处平移画布，拖拽节点标题移动节点位置
- 节点位置自动保存到 localStorage，刷新页面不丢失
- 双击空白处重置视图，自动适应按钮一键居中所有节点
- Bezier 曲线连线，带动画流动效果
- 右下角小地图，点击可快速导航

### 📖 节点弹窗系统

- 点击节点标题弹出详情面板
- 支持所有节点类型：小说信息、大纲、基础设定、角色、世界观、记忆、章节
- 弹窗显示该节点的汇总信息，无需展开节点即可快速浏览

### ✍️ 专业提示词升级

- **大纲提示词**：融合 Blake Snyder《救猫咪》15 节拍表，要求 AI 按专业编剧结构输出
- **章节提示词**：增加场景分解、POV角色、伏笔管理等专业维度
- **正文提示词**：引入海明威"冰山理论"和余华"冷叙事"风格锚定，增加 burstiness（节奏波动）要求
- **摘要提示词**：结构化"故事圣经"格式，包含角色弧线、情绪节拍、伏笔追踪

### 🏗️ 架构优化

- 前端画布逻辑从 `app.js` 解耦到独立的 `canvas-engine.js`
- CSS 增加无限画布专用样式层
- 版本统一升级到 v4.0.0

## v2.3.0 - 深度借鉴开源项目精华 (2026-07-09)

> 基于对 MemGPT/Letta、LangChain Memory、mem0、Novelcrafter、Sudowrite 的深入研究

### 新增功能

#### 🧠 记忆自动整合系统（借鉴 mem0）

- **MemoryConsolidator 类**：当记忆条目超过阈值（默认200条）时自动触发整合
- **衰减机制**：每次整合对所有条目应用 0.95 衰减因子，老记忆逐渐淡化
- **低价值修剪**：自动移除 importance × decay < 2.0 且 access_count < 2 的条目
- **相似条目合并**：基于字符 bigram Jaccard 相似度，> 50% 的同类型条目自动合并
- **整合统计**：返回整合前后的条目数变化、修剪数、合并数

#### 📊 上下文窗口监控（借鉴 MemGPT）

- **TokenEstimator 类**：估算中英文文本的 token 数（中文约 1.5 字/token）
- **上下文大小估算**：`estimate_context_size()` 计算记忆系统总 token 占用
- **自动压缩判断**：`should_compress()` 当上下文超过 max_tokens 时返回 true
- **文本压缩**：`compress_context()` 将文本截断到指定 token 数内

#### 🔍 多因子语义检索（借鉴 LangChain）

- **综合评分算法**：score = relevance_weight × 关键词匹配 + recency_weight × 衰减因子 + importance_weight × 重要度/10
- **可调权重参数**：relevance_weight=0.5, recency_weight=0.2, importance_weight=0.3
- **保持向后兼容**：原有 search() 方法签名不变，新参数均有默认值

#### 🎭 结构化角色卡系统（借鉴 SillyTavern Character Card）

- **createCharacter() 工厂函数**：统一创建包含所有字段的角色对象
- **新增字段**：
  - `speech_style`：说话风格（如"说话简洁，很少用形容词，偶尔冷幽默"）
  - `example_dialogue`：示例对话列表，支持动态添加/删除
  - `current_state`：当前状态（如"正在调查连环失踪案"）
  - `relationships`：结构化关系 {角色名: "关系描述"}，支持动态编辑
  - `first_appearance`：首次出场章节
  - `arc`：角色弧线/成长轨迹
  - `description`：外貌描述（兼容旧字段 appearance）
- **角色编辑器 UI**：全新的模态框编辑界面，支持所有新字段
- **角色信息注入增强**：`buildCharacterPrompt()` 生成完整的结构化角色档案注入提示词
- **数据迁移**：自动将旧数据的 appearance → description，catchphrase → speech_style

#### 🎬 场景级控制（借鉴 Novelcrafter）

- **sceneConfig 每章独立配置**：
  - `characters[]`：出场角色多选（从角色列表中选择）
  - `contextRange`：上下文范围（minimal=2章 / summary=5章 / full=10章）
  - `sceneType`：场景类型（narrative/action/dialogue/transition）
  - `pacing`：节奏控制（slow/normal/fast）
  - `emotionalTone`：情绪基调
  - `sceneGoal`：场景目标
  - `styleOverride`：风格覆盖
- **场景配置 UI**：每章新增可折叠的「🎬 场景配置」面板
- **智能上下文注入**：`generateContent()` 根据 sceneConfig 自动调整：
  - 只注入选中角色的信息（而非全部角色）
  - 根据 contextRange 调整前文章节数
  - 追加场景类型、节奏、情绪、目标等约束到提示词

#### ✅ 写作质量一致性检查（借鉴 Novelcrafter/Sudowrite）

- **checkChapterConsistency() 函数**：五维度一致性检查
  - 角色性格是否 OOC（与设定对比）
  - 设定是否矛盾（与世界观对比）
  - 时间线是否合理
  - 伏笔是否遗忘或重复
  - 角色说话风格是否一致
- **触发方式**：
  - 每章操作栏新增「一致性检查」按钮
  - 右键菜单新增「一致性检查」选项（content 类型）
- **结构化输出**：问题分类列表 + 修改建议

#### 📝 自动事实提取（借鉴 mem0）

- **build_fact_extraction_prompt()**：从章节内容中提取四类关键事实
  - 人物事实：身份、能力、习惯、关系变化
  - 设定事实：世界观规则、地名、势力、物品
  - 剧情事实：已发生的重大事件
  - 伏笔事实：已埋下但未揭示的伏笔
- **自动存入语义索引**：提取的事实通过 HierarchicalMemory.add_fact() 存入长期记忆

### 改进

- `generateContent()` 全面增强：集成场景配置、角色选择、上下文范围控制
- `loadState()` 增强：自动迁移旧数据，补充 sceneConfig 默认值
- 所有新代码保持向后兼容，不影响已有功能
- 更新 README.md 和 BEST_PRACTICES.md 记录新功能

## v2.2.0 - 基于最佳实践的重大更新 (2026-07-09)

### 新增功能

#### 🌐 关键词触发式世界设定（深度借鉴 SillyTavern World Info）

- **保底条目（Constant）**：无需关键词，每次生成都注入，适合世界观基调、核心规则
- **主关键词 + 次级关键词**：支持精确控制触发条件
- **次级关键词逻辑（Selective Logic）**：
  - AND_ANY：主key匹配 + 次级key任意一个匹配
  - NOT_ALL：主key匹配 + 次级key并非全部匹配
  - NOT_ANY：主key匹配 + 次级key全部不匹配
  - AND_ALL：主key匹配 + 次级key全部匹配
- **正则匹配**：关键词支持正则表达式（以 `/` 为定界符，如 `/禁[术法咒]/`）
- **互斥组（Inclusion Group）**：同组条目只激活一个，避免信息过载
- **触发概率（Probability）**：按概率触发（0-100%），增加变化性
- **多匹配源**：扫描大纲、剧情、背景、角色设定、前文章节摘要
- **优先级排序**：数值越大越靠后，影响越大

#### 🔄 多阶段去AI味流水线

- 新增四阶段流水线：风格改写 → 去模板化 → 人味注入 → 最终润色
- 每个阶段可独立开关
- 实时显示处理进度和 AI 味分数变化
- 右键菜单一键执行

#### 🤖 反AI味词汇自动替换系统

- 内置 60+ 个 AI 高频词映射表
- 支持本地快速检测 AI 味分数（0-100 分）
- 一键自动替换为人类化表达
- 右键菜单新增「AI味检测(本地)」选项

### 改进

- 更新 README.md，添加新功能说明
- 更新 BEST_PRACTICES.md，标记已完成的改进项
- 优化右键菜单，新增「🔄 去AI味流水线」子菜单
- 增强状态持久化，保存 World Info 和反 AI 味配置

## 迭代记录（50+轮改进）

### 第1轮：扩展反AI味禁用词库

- 禁用词从13个扩展到40+个，覆盖更多AI写作指纹
- 新增：眉头微蹙、嘴角上扬、眼中闪过一抹、心中暗道、不觉间、霎时间、此刻的他、他深知、无疑、显然、毫无疑问

- 新增"节奏控制"规则：要求场景切换有标点标记，快节奏用短句段落

- 大纲AI评分新增"读者留存"维度：读者看到第几章会弃书？
