## 为什么做这个

市面上的 AI 写作工具有一个通病：**生成的内容 AI 味太重**。

"心中一凛"、"眼中闪过一丝XX"、"嘴角勾起一抹XX"、排比句、段尾总结抒情、形容词堆砌……这些是 AI 写作的指纹，读者一眼就能看出来。

本项目的核心思路：**不是让 AI 写得更多，而是让 AI 写得更像人**。

通过精心设计的提示词体系，从生成阶段就压制 AI 味，配合右键菜单的"AI 味检测"和"去 AI 味重写"功能，反复打磨直到满意。

## ✨ 功能特性

### 创作流程

```
基础设定 → AI 生成大纲 → 章节细纲 → 逐章正文 → 右键打磨 → 导出发布
```

| 功能 | 说明 |
|------|------|
| 📋 **大纲生成** | 基于设定自动生成故事大纲，要求矛盾尖锐、转折有因果，包含黄金三章设计 |
| 📖 **章节拆分** | 大纲自动拆分为章节细纲，每章有明确事件驱动、情绪曲线、钩子强度 |
| ✍️ **正文创作** | 逐章生成正文，内置反AI味引擎v2.0，包含黄金200字开头和结尾钩子 |
| 🖱️ **右键菜单** | 选中文本右键，15项AI操作一键完成 |
| 🤖 **AI 助手** | 右侧面板，随时问问题、构思剧情，自动感知当前章节上下文 |
| 📚 **类型模板** | 预置10种常见网文类型（都市、仙侠、玄幻、系统流、末日、言情、历史、科幻、游戏、悬疑） |
| 💾 **自动保存** | 所有内容自动保存到浏览器，支持导入导出 |
| 🔧 **运行时配置** | 在界面中直接切换模型，无需重启 |
| 📤 **多格式导出** | 支持导出JSON、TXT、Markdown三种格式 |

### 右键菜单（15项操作）

| 操作 | 说明 |
|------|------|
| **AI味检测** | 10项维度检测，逐项打分，0-100分评分 |
| **去AI味重写** | 按禁用词清单 + 结构规则重写选中内容 |
| **压缩精简** | 砍废话，字数减40%信息量不减 |
| **砍形容词** | "温暖的阳光" → "阳光" |
| **加动作写情绪** | 把形容词情绪改成行为描写 |
| **加口语感** | 像讲故事，不像写作文 |
| **加伏笔** | 添加前后呼应的伏笔，标注埋设点和回收点 |
| **改节奏** | 调整快慢节奏，紧张的地方快节奏，重要的地方慢节奏 |
| **加感官描写** | 用视觉/听觉/嗅觉/触觉/味觉的具体细节替代抽象描述 |
| **续写** | 根据上下文自动续写500-1000字 |
| **改写风格** | 改写为热血/悬疑/搞笑/文艺/硬核等不同风格 |

## 🆕 新增功能（v2.0）

### 角色管理
- 添加角色卡片（姓名、外貌、性格、口头禅、背景）
- 角色信息自动注入AI提示词
- 防止角色OOC（性格不一致）

### 世界观笔记
- 记录设定、规则、地名、势力等
- 世界观信息自动注入提示词，保持一致性

### 灵感笔记
- 随时记录灵感、想法、片段
- 支持标签分类
- 可以一键插入正文

### 拆书分析
- 粘贴参考文本，AI分析其写作技巧
- 输出：句式分析、节奏分析、对话技巧、可学习的点
- 支持选择分析重点（全面/句式/节奏/对话/情绪/描写）

### 一致性检查
- AI检查全文中的人名、地名、时间线是否一致
- 检查人物性格是否前后矛盾
- 检查设定是否有冲突

### 读者视角审查
- AI模拟读者视角审读
- 指出可能让读者弃书的地方
- 给出"如果我是读者，我在哪里会跳过"的反馈

### 版本历史
- 每次生成内容自动创建版本快照
- 可以查看历史版本
- 可以回滚到任意版本
- 最多保存20个版本

### 批量生成
- "一键生成全部"按钮，逐章自动生成正文
- 自动等待避免API限流
- 显示总进度

### 写作日历
- 记录每天的写作字数
- 可视化显示写作活跃度
- 激励持续创作

### AI对话搜索
- 可以搜索AI助手的历史对话
- 快速找到之前的建议和灵感

### 快捷键
- `Ctrl+S` — 手动保存
- `Ctrl+E` — 导出JSON
- `Ctrl+/` — 聚焦AI助手
- `Escape` — 关闭弹窗

## 🛡️ 反 AI 味引擎 v2.0

这是本项目的核心差异化能力，经过55轮迭代优化。以下规则已内置在提示词中：

### 🚫 禁用词清单（40+个）

出现以下任何一个，该段直接重写：

```
心中一凛    眼中闪过一丝XX    嘴角勾起一抹XX    一股XX涌上心头
仿佛XX一般  宛如XX            不由得              情不自禁
目光深邃    意味深长           若有所思            恍然大悟
不禁XX      这一刻他明白       他知道，XX          眉头微蹙
嘴角上扬    心中暗道           不觉间              霎时间
此刻的他    他深知             无疑                显然
毫无疑问    不言而喻           与此同时            就在这时
突然间      猛然间             刹那间              恍惚间
不知不觉    本能地             下意识地            条件反射般
如同XX一般  好似XX             恰似XX
```

### 📐 结构规则

- **句子长短交替** — 禁止通篇 15-25 字的"标准句"
- **"的"字密度** — 每段不超过 3 个
- **禁止排比句** — "他感到…他感到…他感到…"直接扣分
- **禁止段尾总结抒情** — "这就是XX的意义啊"全部删掉
- **信息密度** — 每段必须有新信息，禁止原地踏步
- **感官细节** — 用具体感官替代抽象形容

### 💬 对话规则

- 每句对话必须：暴露信息 / 暴露性格 / 推动行动（三选一）
- 不同角色说话方式必须不同
- 用动作代替"XX说道"、"XX淡淡道"
- 要有打断、跑题、答非所问
- 对话要有潜台词

### 🎭 情绪规则

- 用动作和结果表达情绪，不要写"他很愤怒" → 写"他把杯子摔了"
- 禁止大段心理独白
- 让读者从行为中自己感受

### 📖 开头和结尾

- **黄金200字** — 每章开头必须有冲突或悬念
- **结尾钩子** — 每章结尾必须留悬念
- **信息投喂** — 每章至少透露一个新信息

### 📊 AI味检测维度（10项）

1. 禁用词检测
2. 排比句检测
3. 段尾抒情检测
4. 形容词堆砌检测
5. 对话标签检测
6. 心理独白检测
7. "的"字密度检测
8. 句子长度均匀度检测
9. 句式多样性检测
10. 词汇多样性检测

## 🤖 支持模型

所有 OpenAI 兼容格式的 API 均可使用：

| 模型 | Endpoint | 推荐场景 |
|------|----------|----------|
| **DeepSeek** | `https://api.deepseek.com/v1/chat/completions` | 性价比高，中文能力强 |
| **通义千问** | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | 阿里系，稳定 |
| **豆包** | `https://ark.cn-beijing.volces.com/api/v3/chat/completions` | 字节系 |
| **OpenAI** | `https://api.openai.com/v1/chat/completions` | GPT-4o 效果最好 |

配置方式：启动后在界面「模型设置」中填写，或直接编辑 `.env` 文件。
