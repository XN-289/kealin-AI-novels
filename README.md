## 为什么做这个

AI 写小说，快是真快，假也是真假。

"心中一凛"、"眼中闪过一丝XX"、"嘴角勾起一抹XX"、排比三连、段尾抒情总结、形容词不要钱地堆……这些是 AI 写作的指纹。读者扫一眼就知道是机器写的——不是因为写得不好，而是因为**写得太像了**。所有 AI 产出的文本都有一种共同的"正确感"，像流水线上的产品，光滑、均匀、没有毛刺。

但人写的东西不是这样的。人写的东西有偏好、有坏习惯、有突然的灵感和明显的缺陷。**好的小说不是完美的，是有个性的。**

Kealin 的目标就是这件事：

> **用 AI 写，但写出来的东西，人看不出来是 AI 写的。**

这不是一个简单的 prompt 工程问题。它需要从**工具设计、创作流程、提示词架构、文本后处理**四个层面系统性地解决。下面讲我们怎么做的。

## 架构设计

Kealin 不是一个"输入大纲、输出正文"的黑盒。它是一个**完整的创作工作台**，让作者在每一个环节都有控制权，同时让 AI 在底层做脏活累活。

```
┌─────────────────────────────────────────────────────────────────┐
│                        创作者（决策层）                           │
│  设定世界观 → 写大纲 → 拆章节 → 配场景 → 逐章打磨 → 导出        │
└────────────────────────────┬────────────────────────────────────┘
                             │ 每一步都可以介入、修改、推翻
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Kealin 工具层                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ 无限画布  │ │ 角色系统  │ │ 场景控制  │ │ 右键菜单  │          │
│  │ 可视化   │ │ 结构化卡  │ │ 逐章配置  │ │ 20+操作   │          │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ 记忆系统  │ │ 一致性检查│ │ AI味检测  │ │ 导出系统  │          │
│  │ 分层检索  │ │ 五维度   │ │ 10项打分  │ │ 多格式   │          │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    反 AI 味引擎（核心）                           │
│  禁用词过滤 → 结构规则约束 → 对话规则 → 情绪规则 → 后处理重写   │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    大模型 API 层                                  │
│  DeepSeek / 通义千问 / 豆包 / GPT（任意 OpenAI 兼容接口）       │
└─────────────────────────────────────────────────────────────────┘
```

关键设计原则：

- **作者永远是决策者**。AI 是工具，不是作者。每一个生成结果都可以被修改、重写、推翻。
- **控制粒度要细**。不是"写一章"，而是"这一章用这几个角色、这个节奏、这种情绪、这个场景类型"。
- **反馈回路要短**。写完立刻检测、发现问题立刻重写、不用导出到别的工具。

## 反 AI 味引擎

这是 Kealin 的核心。不是调一下 prompt 就完事——是一整套从生成到后处理的流水线。

### 问题诊断：AI 味从哪来？

AI 写作有 10 个维度的"指纹"：

| 维度 | AI 的坏习惯 | 人怎么写 |
|------|------------|---------|
| 禁用词 | "心中一凛"、"不由自主" | 不用这些词 |
| 排比句 | "他感到…他感到…他感到…" | 偶尔用，多数不用 |
| 段尾抒情 | "这就是XX的意义啊" | 不总结，让读者自己感受 |
| 形容词堆砌 | "温暖的、柔和的、金色的阳光" | "阳光"就够了 |
| 对话标签 | "他淡淡道"、"她轻声道" | 用动作代替 |
| 心理独白 | 大段内心OS | 用行为暗示 |
| "的"字密度 | 一段话5个"的" | 一段最多3个 |
| 句子长度 | 通篇15-25字标准句 | 长短交替，有的2字有的50字 |
| 句式 | 主谓宾主谓宾 | 倒装、省略、片段句 |
| 词汇 | 书面语堆砌 | 口语混书面，有个人偏好 |

### 解决方案：四阶段流水线

```
原始文本
  │
  ▼
[阶段1] 禁用词扫描 ──→ 发现任何一个 → 该段标记重写
  │
  ▼
[阶段2] 结构规则检测 ──→ "的"字密度、句长均匀度、排比、段尾抒情
  │
  ▼
[阶段3] 风格改写 ──→ 口语化、打破工整感、加个人痕迹
  │
  ▼
[阶段4] 人味注入 ──→ 不完美的细节、打断、跑题、答非所问
  │
  ▼
输出：读起来像人写的文本
```

一键触发，全自动。也可以只跑其中某一阶段。

### 生成时就按住

后处理是补救，更好的做法是**生成的时候就不让 AI 犯这些毛病**。每个生成请求（大纲、正文、摘要）的提示词里都内置了反 AI 味规则：

- **大纲提示词**：要求"矛盾尖锐、转折有因果、不要善恶对立"
- **正文提示词**：禁用词清单直接贴进去，结构规则写成硬约束
- **摘要提示词**：要求"记录角色状态变化、伏笔埋收、情绪走向"

不是事后洗，是**从源头就不让 AI 写出那些东西**。

## 创作工具链

### 角色系统

借鉴 SillyTavern 的结构化角色卡设计。每个角色不是一段文字描述，而是一个**结构化数据对象**：

- 性格特征、说话风格、示例对话（锚定语言风格）
- 当前状态、关系网络、角色弧线
- 首次出场章节、背景故事、秘密

生成正文时，系统自动把选中角色的完整档案注入提示词。AI 知道这个角色说话是什么风格、当前处于什么状态、跟其他角色是什么关系——**不容易写出 OOC（Out of Character）的内容**。

### 场景控制

借鉴 Novelcrafter 的设计。每章可以独立配置：

| 配置项 | 作用 |
|--------|------|
| 出场角色 | 只注入选中角色，减少干扰 |
| 上下文范围 | 控制注入前文的章节数（2/5/10章） |
| 场景类型 | narrative / action / dialogue / transition |
| 节奏 | slow / normal / fast |
| 情绪基调 | 自由输入："紧张压抑"、"轻松温馨" |
| 场景目标 | "揭露反派身份"、"男女主重逢" |

不是"帮我写一章"，是"帮我写一章节奏快、情绪紧张、只有这两个角色出场的打斗场景"。**控制越细，输出越可控。**

### 记忆系统

借鉴 MemGPT/Letta、LangChain Memory、mem0 的核心设计。AI 写长篇最大的问题是"写着写着就忘了前面写过什么"。Kealin 用分层记忆解决：

```
┌─────────────────────────────────────────────┐
│              上下文窗口（有限）                │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐  │
│  │ 永久记忆   │ │ 章节摘要   │ │ 语义检索   │  │
│  │ 作者手动   │ │ 自动滑窗   │ │ 按需注入   │  │
│  └───────────┘ └───────────┘ └───────────┘  │
└─────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│              外部存储（无限）                  │
│  章节摘要库 · 角色档案库 · 世界观设定库       │
└─────────────────────────────────────────────┘
```

- **永久记忆**：作者手动添加的核心设定，每次都注入
- **章节摘要**：每章自动生成结构化摘要（事件、角色变化、伏笔、结尾状态），滑动窗口最近10章
- **语义索引**：TF-IDF 索引，按相关性检索历史记忆
- **记忆整合**：超过阈值时自动衰减、修剪、合并（借鉴 mem0）

### 一致性检查

五维度自动检测：

- **角色一致性**：性格是否 OOC、说话风格是否偏离
- **设定冲突**：世界观规则是否被违反
- **时间线**：事件顺序是否合理
- **细节一致**：前后文外貌、物品、数字是否矛盾
- **伏笔追踪**：已埋伏笔是否遗忘

### 右键菜单（20+ 操作）

选中文本，右键，一键操作。不需要导出到别的工具再处理。

正文打磨：AI 味检测、去 AI 味重写、压缩精简、砍形容词、加动作写情绪、加口语感、加伏笔、改节奏、加感官描写、续写、改风格……

大纲操作：AI 评分、加强冲突、加因果链、修人物动机、砍废话、加暗线、砍冗余人物、加黄金三章……

## 无限画布

传统的章节列表是线性的——一章接一章，像文档目录。但小说创作不是线性的。角色有支线，情节有分支，伏笔有埋设和回收，场景有并行。**线性的列表看不到这些关系。**

Kealin 的无限画布用一种**可视化链路**的方式呈现创作结构：

- **每个章节是一张卡片**，可以自由拖拽、排列
- **卡片之间有连线**，显示章节之间的顺序关系和自定义连接
- **非章节元素**（角色、剧情线、世界观设定）也可以作为卡片挂载到画布上
- **框选 + 批量操作**：Ctrl+拖拽框选多个卡片，批量删除、批量生成
- **缩放 + 平移**：滚轮缩放，拖拽平移，适配任意规模的小说

画布的价值不是"好看"，是**让创作者看到结构**。当你把所有章节卡片铺开，用线连起来，你能一眼看到：

- 哪条支线断了（没有后续章节的卡片）
- 哪里节奏太快（连续几张卡片都是同一个场景类型）
- 哪个角色出场太少（角色卡片连到的章节卡片太少）
- 伏笔线在哪里（自定义连接标记了伏笔的埋设和回收）

**创作不是从 A 到 B 的直线。画布让这条弯路变得可见、可改。**

## 快速开始

```bash
# 克隆
git clone git@github.com:XN-289/kealin-AI-novels.git
cd kealin-AI-novels

# 装依赖
pip install -r requirements.txt

# 配置 API Key
cp .env.example .env
# 编辑 .env，填入你的 Key

# 启动
python app.py
```

浏览器打开 `http://localhost:20000`，开写。

支持所有 OpenAI 兼容格式的 API：DeepSeek、通义千问、豆包、GPT-4o。启动后在界面「模型设置」里配置，或直接改 `.env`。
