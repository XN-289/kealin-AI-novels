## 为什么做这个

AI 写小说，快是真快，假也是真假。

"心中一凛"、"眼中闪过一丝XX"、"嘴角勾起一抹XX"、排比三连、段尾抒情总结、形容词不要钱地堆……这些是 AI 写作的指纹，读者扫一眼就知道是机器写的。

**Kealin 的活儿就一个：把这股味洗掉。**

不是让 AI 写得更多，是让 AI 写得像人。从提示词层面就把 AI 味按住，配合右键菜单的检测和重写功能，反复打磨，直到以假乱真。

## 🧠 分层记忆系统

> 借鉴 MemGPT/Letta、LangChain Memory、mem0 的核心设计

### 记忆架构

```
┌─────────────────────────────────────────────────────────┐
│                    上下文窗口（有限）                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  永久记忆    │  │  章节摘要    │  │  语义检索    │     │
│  │ 作者手动设定 │  │ 最近N章摘要  │  │ 按需注入     │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│                    外部存储（无限）                        │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  章节摘要库  │  │  角色档案库  │  │  世界观设定库│     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
└─────────────────────────────────────────────────────────┘
```

### 记忆层次

| 层次 | 来源 | 生命周期 | 说明 |
| ---- | ---- | -------- | ---- |
| **永久记忆** | 作者手动添加 | 永久 | 核心设定、规则、要求，每次生成都注入 |
| **章节摘要** | 自动生成 | 滑动窗口（最近10章） | 结构化摘要：事件、角色变化、伏笔、结尾状态 |
| **语义索引** | 自动提取 | 长期（衰减+整合） | 关键事实的 TF-IDF 索引，按相关性检索 |
| **关键词触发** | 作者配置 | 按需 | World Info 式关键词匹配，精准注入世界设定 |

### 记忆自动整合（借鉴 mem0）

当记忆条目超过阈值时自动触发：

1. **衰减** — 所有条目乘以 0.95 衰减因子，老记忆逐渐淡化
2. **修剪** — 移除低价值条目（重要度 × 衰减 < 2.0 且很少被访问）
3. **合并** — 内容相似度 > 50% 的同类型条目自动合并

### 多因子语义检索（借鉴 LangChain）

检索时综合三个因子评分：

```
score = 0.5 × 关键词匹配 + 0.2 × 衰减因子 + 0.3 × 重要度
```

### 上下文窗口监控（借鉴 MemGPT）

- TokenEstimator 估算中英文 token 数
- 自动判断上下文是否需要压缩
- 超限时自动截断低优先级内容

### 角色信息注入

生成正文时，系统自动将选中角色的完整档案注入提示词，包括：

- 性格特征和说话风格（防止 OOC）
- 示例对话（锚定语言风格）
- 当前状态（保持剧情连贯）
- 关系网络（维护角色互动一致性）
- 角色弧线（驱动成长轨迹）

## 🎬 场景级控制

> 借鉴 Novelcrafter 的场景配置设计

场景是 AI 写作的最佳控制粒度 —— 比章节更细、比段落更有语义完整性。

### 每章可独立配置

| 配置项 | 选项 | 说明 |
| ------ | ---- | ---- |
| **出场角色** | 从角色列表多选 | 只注入选中角色的信息，减少干扰 |
| **上下文范围** | minimal / summary / full | 控制注入前文的章节数（2/5/10章） |
| **场景类型** | narrative / action / dialogue / transition | 影响生成风格 |
| **节奏** | slow / normal / fast | 紧张处快刀斩，重要处慢镜头 |
| **情绪基调** | 自由输入 | 如"紧张压抑"、"轻松温馨" |
| **场景目标** | 自由输入 | 如"揭露反派身份"、"男女主重逢" |
| **风格覆盖** | 自由输入 | 覆盖全局风格设置 |

### 智能上下文注入

`generateContent()` 根据场景配置自动调整：

- 只注入选中角色的结构化档案
- 根据上下文范围调整前文章节数
- 追加场景类型、节奏、情绪、目标等约束到提示词

## ✅ 写作质量一致性检查

> 借鉴 Novelcrafter/Sudowrite 的质量控制理念

### 五维度检查

| 维度 | 检查内容 |
| ---- | -------- |
| **角色一致性** | 性格是否 OOC、说话风格是否偏离设定 |
| **设定冲突** | 世界观规则是否被违反、地名/势力名是否一致 |
| **时间线** | 事件顺序是否合理、是否有时间矛盾 |
| **细节一致** | 前后文细节是否矛盾（外貌、物品、数字等） |
| **伏笔追踪** | 已埋伏笔是否遗忘、是否重复提及已回收的伏笔 |

### 触发方式

- 每章操作栏的「一致性检查」按钮
- 正文右键菜单的「一致性检查」选项

## 🌐 关键词触发式世界设定

> 深度借鉴 SillyTavern World Info 设计

当文本中出现特定关键词时，自动注入对应的世界设定。

### 触发特性

| 特性 | 说明 |
| ---- | ---- |
| **保底条目** | 无需关键词，每次生成都注入，适合世界观基调 |
| **主关键词 + 次级关键词** | 精确控制触发条件 |
| **次级关键词逻辑** | AND_ANY / NOT_ALL / NOT_ANY / AND_ALL 四种模式 |
| **正则匹配** | 关键词支持正则表达式（以 `/` 为定界符） |
| **互斥组** | 同组条目只激活一个，避免信息过载 |
| **触发概率** | 按概率触发（0-100%），增加变化性 |
| **多匹配源** | 扫描大纲、剧情、背景、角色、前文章节摘要 |
| **优先级排序** | 数值越大越靠后，影响越大 |

### 示例

- 规则名称：魔法系统
- 触发关键词：魔法、法术、咒语、魔力、元素
- 次级关键词：战斗、施法、修炼（AND_ANY 模式）
- 世界设定：本世界魔法分为五系：火、水、风、土、暗……

## 🖱️ 右键菜单（20+ 项）

选中文本右键，一键操作：

### 正文打磨

| 操作 | 干嘛用 |
| ---- | ------ |
| **AI 味检测** | 10 项维度打分，0-100，哪里假一目了然 |
| **去 AI 味重写** | 按禁用词清单 + 结构规则重洗一遍 |
| **压缩精简** | 砍废话，字数减 40% 信息量不丢 |
| **砍形容词** | "温暖的阳光" → "阳光" |
| **加动作写情绪** | "他很愤怒" → "他把杯子摔了" |
| **加口语感** | 像讲故事，不像写作文 |
| **加伏笔** | 埋线 + 标注回收点 |
| **改节奏** | 紧张处快刀斩，重要处慢镜头 |
| **加感官描写** | 视觉/听觉/嗅觉/触觉/味觉，具体细节替代抽象形容 |
| **续写** | 接着往下写 500-1000 字 |
| **改风格** | 热血 / 悬疑 / 搞笑 / 文艺 / 硬核 |
| **一致性检查** | 五维度质量检查 |

### 去 AI 味流水线

| 模式 | 说明 |
| ---- | ---- |

| **仅人味注入** | 只增加口语化表达 |
| **AI 味检测(本地)** | 不调用 API，本地快速检测并自动替换 |

### 大纲 / 章节操作

| 操作 | 干嘛用 |

| **修人物动机** | 找到真实动机 |

## 🛡️ 反 AI 味引擎

Kealin 的核心。57 轮迭代，专洗 AI 味。

### 🚫 禁用词（40+个）

出现任何一个，该段直接重写：

```
心中一凛    眼中闪过一丝XX    嘴角勾起一抹XX    一股XX涌上心头
仿佛XX一般  宛如XX            不由得              情不自禁
目光深邃    意味深长           若有所思            恍然大悟
不禁XX      这一刻他明白       他知道，XX          眉头微蹙
嘴角上扬    心中暗道           不觉间              霎时间
此刻的他    他深知             无疑                显然
毫无疑问    不言而喻           与此同时            就在这时
突然间      猛然间             刹那间              恍惚间
不知不觉    本能地             下意识地            条件反射般
如同XX一般  好似XX             恰似XX
```

### 📐 结构规则

- **长短交替** — 禁止通篇 15-25 字的标准句
- **"的"字密度** — 每段不超过 3 个
- **禁排比** — "他感到…他感到…他感到…"直接扣分
- **禁段尾抒情** — "这就是XX的意义啊"全删
- **信息密度** — 每段必须有新东西，禁止原地踏步
- **感官细节** — 具体的替代抽象的
- **段落不规则** — 有的 2 句，有的 5 句，避免工整感

### 💬 对话规则

- 每句对话必须：暴露信息 / 暴露性格 / 推动行动（三选一）
- 不同角色说话方式必须不同
- 用动作代替"XX说道"
- 要有打断、跑题、答非所问
- 对话要有潜台词

### 🎭 情绪规则

- 用动作和结果表达，不要写"他很愤怒"
- 禁止大段心理独白
- 让读者从行为中自己感受

### 📖 开头和结尾

- **黄金 200 字** — 开头必须有冲突或悬念
- **结尾钩子** — 每章结尾必须留悬念
- **信息投喂** — 每章至少透露一个新信息

### 📊 AI 味检测（10 项维度）

禁用词 · 排比句 · 段尾抒情 · 形容词堆砌 · 对话标签 · 心理独白 · "的"字密度 · 句子长度均匀度 · 句式多样性 · 词汇多样性
